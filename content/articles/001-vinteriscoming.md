Over the past three weeks or so, [Nils Daumann](http://slindev.com/ "Nils Daumann's blog") and I sat down together and wrote a shader based 2D graphics engine for iOS and Mac OS X. First of all, we do know about Cocos 2D, but we are both not really fans of it. Mostly because it didn't supported Shaders at the time we looked into it (although that changed now with Cocos 2D 2.0, but thats still in alpha) and it uses this stupid coordinate system from OpenGL (ES). But the biggest contra is that its extremely bloated and hard to get into, even though it has an awesome community and tons of documentation. However, it has too many features and most of them aren't really needed most of the time.

So, we did it a little bit different: Vinter is written in C++, it is completely shader based and it uses the left hand coordinate system known from iOS. It works on iOS and Mac OS X but we are pretty confident that it will also work on Android with some minor changes mostly related to image loading and file handling (typical platform specific stuff). Like Cocos2D, Vinter is completely open source and can be found on [GitHub](https://github.com/JustSid/Vinter/ "Vinter on Github").

While we designed Vinter, we made some radical decisions, the biggest is that we won't create any kind of UI controls for it. Simply put, you will never find a button or textfield class in our official repository! The reason is actually quite simple, the Cocoa and Cocoa-Touch text rendering system is AWESOME! Seriously, I have never seen text rendered this crisp and clear on any other device. We want to keep it that way and don't destroy the experience with bitmap fonts or slow and horrible looking TTF rendering. If you want to render a text, use a NSTextField or UILabel!

Probably the second biggest decision we made was, that we don't support OpenGL ES 1.1! We only support shader based rendering, because this is how modern GPUs render. If you use the FFP on a Mac or a modern iOS device (read, everything from the third generation and all iPads), it will be emulated with shaders which is slow and also you are kinda very limiting. We don't want this, so we give you full freedom with shaders. Shaders aren't the future, neither are they bleeding edge, they are state of the art! So what we do is supporting OpenGL ES 2.0, OpenGL 2.0 (GSlang 2.0) and OpenGL 3.2 (GSlang 1.50) which runs on every device shipped in the last three years.

If you are interested in Vinter, you can find the source code together with a sample project at the already mentioned [GitHub](https://github.com/JustSid/Vinter/ "Vinter on Github") page. You can also take a look into the [Vinter Viki](http://vinter2d.org/ "Vinters Wiki") we are currently building (be aware, the content is a little bit limited at the moment).
